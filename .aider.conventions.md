# Aider conventions

## Context search

Before starting a task, search for relevant context from the knowledge vault:

```
/run rg "keyword" /workspaces/ai_home/obsidian/
```

Useful vault paths:
- `/workspaces/ai_home/obsidian/1-projects/` — active projects
- `/workspaces/ai_home/obsidian/2-areas/` — ongoing areas of responsibility
- `/workspaces/ai_home/obsidian/3-resources/` — reference material

Add a relevant note as read-only context:
```
/read /workspaces/ai_home/obsidian/1-projects/some-note.md
```

## File search

Find files in the project:
```
/run fd <pattern>
/run rg <symbol>
```

## Code style

- Follow conventions already present in the file being edited
- No comments unless the WHY is non-obvious
- Minimal diffs — only change what the task requires
- No auto-commits — always confirm before committing

## Context budget

Running on local 12–32B models. Keep searches focused:
- Use `rg -l` to find which files match before reading them
- Pipe long output: `rg "keyword" /workspaces/ai_home/obsidian/ | head -20`
- Add only the most relevant files to context, not whole directories
